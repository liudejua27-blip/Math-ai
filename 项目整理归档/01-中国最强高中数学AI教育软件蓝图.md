# 高中数学思维训练平台蓝图

整理日期：2026-06-06

## 1. 产品定位

产品暂定名：`Math-SEARAG Learning Agent / 高中数学思维训练平台`

目标不是做一个“答案生成器”，而是做一个能够承担真实辅导任务的引导式学习 Agent：

- 看得懂题：识别文字、公式、条件、图形、参数范围和隐含约束。
- 看得懂学生：读取学生自己的步骤，而不是只讲标准答案。
- 找得到断点：定位第一处错误或第一处不严谨推理。
- 问得出关键：根据诊断结果决定追问、提示、订正、变式或人工复核。
- 讲得清思路：用文字、公式、图上标注、动画和 Geometry Lab 解释为什么这样想。
- 练得出能力：把错因转成可训练的能力原子，并生成同因变式题。
- 记得住成长：形成学生长期数学思维档案，而不是一次性答题。

## 2. 三条产品硬规则

```text
没有学生步骤，不做错因诊断。
有学生步骤，先找第一断点。
不先给完整答案，先给追问、纠偏、订正和迁移训练。
```

## 3. 中国高中数学场景的真正痛点

| 用户 | 痛点 | 新版产品回答 |
| --- | --- | --- |
| 高一学生 | 概念刚学，图形和函数关系抽象 | 用追问、微脚手架和动态讲解把概念过程可视化 |
| 高二学生 | 导数、圆锥曲线、立体几何开始断层 | 用结构化步骤诊断第一处断点 |
| 高三学生 | 错题多，复盘慢，解析看懂但再做仍错 | 每题输出错因原子、同因变式和迁移检测 |
| 家长 | 不知道孩子数学弱在哪里 | 用 LearnerMemory 生成非焦虑化学习报告 |
| 老师 | 批量诊断成本高 | 班级错因热力图、题型薄弱点、作业推荐 |
| 教培机构 | 个性化辅导依赖名师，规模化难 | Learning Agent 诊断引擎 + 教师审核后台 |

## 4. 差异化功能矩阵

| 能力 | 普通搜题 | Math-SEARAG Learning Agent |
| --- | --- | --- |
| 答案讲解 | 给标准解析 | 先判断学生从哪一步开始错 |
| 教学方式 | 直接讲答案 | SocraticPolicy 决定追问、提示、订正、变式 |
| 长期记忆 | 基本没有 | AtomMemory、TopicMemory、StrategyMemory |
| 图形讲解 | 静态图或文字描述 | 图上标注、证据确认、GeometrySceneSpec |
| 立体几何 | 多数只给文字 | Geometry Lab：正方体/三棱锥关卡，后续接 Three.js/R3F |
| 错因分析 | 粗略归因 | 错因原子：审题、定义域、分类讨论、数形结合、空间想象等 |
| 训练推荐 | 随机相似题 | 同错因、同结构、变参数、跨题型迁移训练 |
| 可信度 | 容易幻觉 | evidence_id、VerifierTrace、人机确认、低置信度降级 |

## 5. MVP 选择

第一版必须收窄，不要铺太大。

P0 工程能力：

- `SocraticPolicyEngine`：无步骤请求步骤，有步骤先首错追问。
- `VerifierTraceEngine`：关键数学 claim 必须有验证路径。
- `LearnerMemoryEngine`：记录错因复发率、迁移率、自我修正率。
- 诊断卡显示第一断点、验证路径、追问、订正卡和变式题。

P0 题型：

- 函数性质：定义域、单调性、奇偶性、零点。
- 导数应用：切线、单调区间、极值、恒成立、参数讨论。
- 立体几何：正方体、三棱锥、线面角、二面角、空间距离。

P1 能力：

- `StructuredFormulaEvidenceEngine`
- `VisualEvidencePipeline`
- `MathCognitiveGraphEngine`
- 学生画像页
- 低置信度公式/图形确认 UI

P2 能力：

- Geometry Lab 深化。
- GSAP 订正卡动画。
- Three.js/R3F 立体几何渲染。
- 老师后台和家长报告。

暂不做：

- 全试卷自动批改。
- 全学科覆盖。
- 作文式开放问答。
- 自动承诺提分。
- 完全无人审核的高风险结论。

## 6. 商业价值路径

### 阶段 A：C 端高三错题复盘

卖点：每天 3 道错题，10 分钟找出真正断点，并用同因变式验证是否迁移。

收费方式：

- 免费：每日 1 题基础诊断。
- 会员：无限错题复盘、错因档案、同因变式。
- 专项包：导数、立体几何、解析几何冲刺包。

### 阶段 B：学生画像与家长报告

卖点：家长不看复杂题，只看孩子弱点和下一周行动。

报告原则：

- 不制造焦虑。
- 不排名羞辱。
- 不承诺提分。
- 给出可执行训练建议。

### 阶段 C：老师与教培机构

卖点：把“批改”和“个性化作业推荐”规模化。

功能：

- 班级错因热力图。
- 学生错因分组。
- 同因变式作业包。
- 老师审核与二次编辑。
- 教研题库沉淀。

## 7. 核心护城河

1. 数学证据结构：MPES 把题干、公式、图形、步骤、验证、错因统一成可审计数据。
2. SocraticPolicy：根据诊断结果决定怎么教，而不是固定输出解析。
3. VerifierTrace：关键数学 claim 有验证路径。
4. LearnerMemory：持续记录错因原子复发率、迁移率、自我修正率。
5. 图上讲解引擎：不是生成一段文字，而是生成可交互讲解。
6. 题型工作流：每类题有独立策略、验证规则和图形模板。
7. 教学安全边界：未成年人数据保护、低置信度降级、教师/家长可审计。

## 8. 一年路线

| 时间 | 目标 | 验收 |
| --- | --- | --- |
| 0-2 周 | SocraticPolicy + VerifierTrace | 无步骤不诊断；失败 claim 有 trace |
| 3-4 周 | LearnerMemory | 能显示 A07 复发率、迁移率、推荐训练 |
| 5-6 周 | FormulaEvidence + VisualEvidence | 低置信度公式/几何关系可确认 |
| 7-8 周 | MathCognitiveGraph 2.0 | 能解释为什么推荐变式或 Geometry Lab |
| 9-12 周 | 100 题 pilot + 30 人内测 | FWSA、Atom Accuracy、Self-Repair、Transfer、Teacher Agreement 可统计 |
| 4-6 月 | 学生端产品化 | 错因画像和复习计划稳定 |
| 7-9 月 | 家长报告和老师后台 | 班级错因热力图可用 |
| 10-12 月 | 商业试点 | 教培机构/学校小规模付费试点 |

# 3D 几何、游戏化与图上讲解方案

## 0. 当前优先级

Geometry Lab 是强差异化能力，但在 Learning Agent 2.0 中属于第三优先级。它必须建立在以下能力之上：

1. `VisualEvidencePipeline`：几何关系必须先变成可确认 evidence。
2. `VerifierTraceEngine`：关键几何 claim 必须有验证路径或人工复核状态。
3. `SocraticPolicyEngine`：决定何时进入 Geometry Lab，而不是每题都强推 3D。
4. `LearnerMemoryEngine`：几何关卡结果必须回写错因原子、迁移率和空间推理画像。

第一版不做开放式 3D 游戏，先做受控 `GeometrySceneSpec` + 正方体/三棱锥关卡。

## 1. 为什么几何是核心差异化

中国高中数学里，立体几何和解析几何经常不是“不会算”，而是“看不见关系”。如果产品能把空间关系、辅助线、截面、投影、夹角和距离做成可交互讲解，就会明显区别于普通搜题和聊天式 AI。

## 2. 三层可视化体系

| 层级 | 技术 | 用途 |
| --- | --- | --- |
| 图上标注 | HTML + SVG + Canvas | 题图高亮、辅助线、公式旁注、步骤箭头 |
| 动态讲解 | GSAP 时间轴 | 分步骤出现、颜色变化、推导过程动画 |
| 3D 几何实验室 | SVG 投影 P0；Three.js / React Three Fiber P2 | 正方体、三棱锥、球、圆锥、截面、旋转视角 |
| 高质量视频 | Manim | 生成课程级讲解短视频、知识点动画 |
| 数学构造工具 | GeoGebra API | 平面几何、函数图像、动态参数拖动 |

## 3. 立体几何 MVP 模板

第一批规划 6 个模板，但 P0 只做正方体和三棱锥，不要一开始追求无限自由建模。

1. P0 正方体：线线角、线面角、二面角、空间距离、截面。
2. P0 三棱锥：高、底面投影、二面角。
3. P1 长方体：坐标法、体对角线、投影。
4. P1 四棱锥：侧面关系、截面。
5. P2 球与多面体：外接球、内切球。
6. P2 圆锥/圆柱：轴截面、母线、侧面积。

每个模板都要有：

- 标准顶点命名。
- 可旋转相机。
- 辅助线/辅助面开关。
- 分步骤讲解 timeline。
- 截图导出。
- 绑定题目 evidence_id。
- 绑定 VerifierTrace 或 human_review 状态。
- 完成后回写 LearnerMemory。

## 4. 图上讲解卡

每道题输出一张或一组可保存讲解卡：

- 左侧：题图/函数图/3D 截图。
- 右侧：当前步骤解释。
- 图上：高亮本步涉及的点、线、面、角、距离。
- 底部：错因原子和下一次提醒。

示例：

```json
{
  "card_title": "本题第一处断点：漏看定义域",
  "visual": {
    "type": "function_graph",
    "highlight": ["domain_x_gt_0", "critical_point_x=e^(a-1)"]
  },
  "explanation": "你直接求极值，但没有先确认 x>0。导数题中定义域会决定端点和临界点是否有效。",
  "next_prompt": "下次遇到含 ln x 的函数，第一步先写定义域。"
}
```

## 5. 游戏化不是装饰，而是训练结构

不要做轻飘飘的积分/徽章系统。游戏化必须服务数学能力。

推荐设计：

- 能力地图：函数、导数、解析几何、立体几何、数列、不等式。
- 关卡：每个错因原子是一组关卡。
- Boss 题：综合多个错因原子的高考压轴题。
- 3D 探索关：让学生旋转几何体、找到辅助线或截面。
- 连胜奖励：只奖励“同因变式连续做对”，不奖励刷题数量。
- 复活机制：做错后进入低门槛 scaffold 题。

## 6. 3D 游戏方向

可以把立体几何做成“空间推理训练场”，但不要把整个产品做成游戏。推荐作为独立模块：

- `Geometry Lab`：学生可拖动旋转几何体。
- `Find the Plane`：根据题目找截面。
- `Angle Hunter`：找线线角、线面角、二面角。
- `Projection Quest`：把空间问题转成平面投影。
- `Coordinate Builder`：选择坐标原点和坐标轴，训练坐标法。

每个游戏关卡都必须回流到学习档案：

- 学生是否会找辅助面。
- 是否能识别垂直和平行关系。
- 是否依赖坐标法。
- 空间想象弱点是哪一类。
- 是否能迁移到同因变式题。
- 是否通过追问完成自我修正。

## 7. 渲染安全边界

- LLM 不直接执行任意 JS。
- LLM 只输出受控 scene spec。
- 前端只渲染白名单动作。
- 用户上传图片要脱敏与限制大小。
- 生成的讲解图要保留来源题目、模型版本、证据编号。

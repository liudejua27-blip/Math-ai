# 当前权威总纲：Math-SEARAG Learning Agent

日期：2026-06-06

## 1. 战略迁移

当前项目已经从：

> Math-SEARAG 诊断系统

升级为：

> **Math-SEARAG Learning Agent：面向高中数学错题诊断的可验证、可视化、个性化引导式学习系统。**

它不是“拍照搜题”，也不是“直接给答案”的聊天机器人。核心价值是把学生错误转化为可定位、可验证、可训练、可长期追踪的思维对象。

## 2. 三条产品硬规则

```text
没有学生步骤，不做错因诊断。
有学生步骤，先找第一断点。
不先给完整答案，先给追问、纠偏、订正和迁移训练。
```

## 3. 产品闭环

```text
学生上传题目/输入题目
        ↓
系统要求输入学生步骤
        ↓
结构化公式/图形证据
        ↓
低置信度字段确认
        ↓
首错定位 + 严格门禁
        ↓
VerifierTrace 检查关键数学 claim
        ↓
SocraticPolicyEngine 决定教学方式
        ↓
学生被引导自我修正
        ↓
生成订正卡和思维图谱
        ↓
生成同因变式
        ↓
检测迁移是否成功
        ↓
更新 LearnerMemory
        ↓
推荐下一题 / Geometry Lab / 老师复核
```

这是真正的“诊断 - 追问 - 订正 - 迁移 - 画像”闭环。

## 4. Learning Agent 2.0 七个引擎

| 引擎 | 责任 |
| --- | --- |
| `SocraticPolicyEngine` | 根据诊断结果决定追问、提示、订正卡、变式、Geometry Lab 或人工复核 |
| `LearnerMemoryEngine` | 记录错因复发率、迁移率、自我修正率、掌握度和周计划 |
| `VisualEvidencePipeline` | 将题图/几何图/函数图识别结果转成可确认 evidence |
| `VerifierTraceEngine` | 为求导、等价、代入、定义域、几何关系等关键 claim 生成验证路径 |
| `MathCognitiveGraphEngine` | 生成 Problem Graph、Reasoning Graph、Learner Graph |
| `StructuredFormulaEvidenceEngine` | 管理 LaTeX、公式树、空间关系、低置信度 token 和用户确认 |
| `RemediationLoopEngine` | 负责诊断、追问、自我修正、同因变式、迁移检测和画像更新 |

## 6. 技术路线

- TypeScript-first：产品主控、agent/tool calling、workflow、策略、记忆、错因原子、图谱协议和 UI 全部向 TypeScript 收敛。
- Python verifier：Python 只保留数学验证、OCR、符号计算、几何计算等工具角色。
- VerifierTrace：关键数学 claim 必须绑定 verifier、status、evidenceIds、failureReason 和 confidence。
- Evidence confirmation：视觉/公式低置信度证据必须先确认，再进入正式诊断。
- GeometrySceneSpec：所有几何讲解先通过白名单 scene spec，不让模型直接生成任意 JS。

## 7. 论文路线

论文主线升级为：

> A Visual, Verifiable, and Personalized Thinking Agent for High-School Mathematics Error Diagnosis with Structured Evidence and Misconception-Atom Memory.

中文题目建议：

> 面向高中数学错题诊断的可视化可验证个性化思维 Agent：基于结构化数学证据、视觉证据链与错因原子记忆的研究。

核心创新点：

1. MPES：结构化数学证据表示。
2. Diagnosis-conditioned SocraticPolicyEngine。
3. VerifierTrace：可验证数学诊断链。
4. Misconception-atom-centered LearnerMemory。
5. MathCognitiveGraph：可追溯数学认知图谱。
6. VisualEvidencePipeline：视觉证据确认机制。
7. 同因变式迁移率评估。

## 8. 商业路线

推荐商业切入顺序：

1. C 端学生错题复盘：先证明单次诊断、自我修正和同因变式训练有效。
2. 学生画像页：展示错因复发率、同因迁移率、自我修正率和本周训练计划。
3. 家长报告：把错因画像和复盘成果转为可理解的成长报告。
4. 老师后台：班级错因热力图、题型薄弱点、同因训练布置。
5. 教培机构 SaaS：在数据、标注、内容模板和合规成熟后再推进。

# 项目整合升级执行记录：DeepSeek 工作台 + 可验证 Learning Agent + 几何可视化

日期：2026-06-07

## 本轮定位

项目主体固定为：

> Math-SEARAG Learning Agent：面向高中数学思维诊断的深度学习工作台。

它不是普通聊天机器人，也不是拍照搜题工具。核心闭环是：

```text
学生题目与步骤 -> 首错定位 -> 严格门禁 -> VerifierTrace -> SocraticPolicy -> 订正卡 -> 同因变式 -> 学生画像 -> Geometry Lab 推荐
```

## 前端整合

前端采用 DeepSeek GUI 式工作台，但不复制 Electron 代码。

- 左侧：学生画像、错因原子、训练计划、题型入口。
- 中间：保留 Vercel AI Chatbot 的对话、工具调用、artifact 和输入体验。
- 右侧：Agent Inspector，展示第一错步、严格门禁、验证链、画像更新、训练计划和几何实验推荐。
- 工作台事件协议用于展示 agent 过程，而不是让 UI 自己猜测诊断状态。

## 后端/Agent 整合

- TypeScript/Vercel AI SDK workflow 是主控，负责 agent、tool calling、错因规则、教学策略、图谱协议和 UI。
- Qwen-Agent 作为后端 agent/tool 基座参考，用于后续 adapter，不替代当前 TS workflow。
- Python 只保留为 verifier/toolbox，负责 SymPy、OCR、几何计算和数值采样。
- DeepTutor 参考学习记忆、skills、knowledge base 和 visualize/animator 思路。
- MathCrew 只参考错题闭环与 problem bank 思路，不复制代码。

## 几何模块整合

几何模块不是娱乐游戏，而是借鉴游戏创作的 3D 场景、镜头、任务反馈、步骤动画和空间交互。

- 当前仍使用白名单 `GeometrySceneSpec`。
- target 必须绑定 `correctRefs` 和 `evidenceIds`。
- 禁止任意 `script`、`code`、`javascript` 字段。
- A34 等几何错因出现时，右侧 Inspector 推荐 Geometry Lab。
- 后续可用 Three.js / React Three Fiber 渲染立体几何，但模型只能产出结构化 scene spec。

## 研究假设

| 编号 | 假设 |
| --- | --- |
| H1 | 相比 Naive OCR + Text RAG，公式感知结构化证据增强 RAG 能提高答案准确率和证据 grounding 水平。 |
| H2 | 结构化证据对公式密集题、几何/图形题和多步推导题收益高于普通文字题。 |

## 核心贡献

### 贡献 1：公式感知 Human-in-the-loop Parsing

数学输入不只是文字，需要识别公式、上下标、分式、根号、区间、图形线索和学生步骤。低置信度字段必须用户确认，避免错误 OCR 污染后续推理。

### 贡献 2：MPES 数学证据 Schema

将题目拆成：

- C：题干条件
- F：公式
- G：图形线索
- S：学生步骤
- A：知识/错因原子
- V：验证结果
- P：正确路径
- E：错误定位

每个对象带 ID、来源、置信度、位置和关系边。

### 贡献 3：自适应检索路由

| 题型 | 路径 |
| --- | --- |
| 普通文字题 | Text RAG + 条件匹配 |
| 公式密集题 | LaTeX / Formula Retrieval |
| 几何图形题 | 图形线索 + 关系边检索 |
| 多步推导题 | Selective Graph Retrieval |
| 高风险答案 | 生成后验证 + 低置信度或拒答 |

### 贡献 4：符号验证与证据验证

LLM 负责理解、规划、解释；符号系统负责计算与验证。

验证内容：

- formula grounding
- condition usage
- expression equivalence
- derivative check
- equation substitution
- reasoning path support
- variant problem answer check

### 贡献 5：错因原子诊断

把学生错误映射到高考数学原子能力图谱，而不是只给章节标签。

### 冲 Top A 的补强要求

若目标是 CCF-A/Top A，必须具备：

1. 300-500+ 题主测试集，最好有外部泛化集。
2. 公开或可审计的 MPES 标注规范。
3. 强基线：Closed-book VLM、OCR+Text RAG、OCR+LaTeX RAG、Structured RAG、Graph/Hybrid、Full。
4. 消融实验证明每个模块贡献。
5. 统计显著性和置信区间。
6. 失败案例和成本分析。
7. 更强原创算法，例如可学习公式图检索器、符号约束解码器、验证驱动路由器。

## 论文成功标准

第一阶段成功：

- 50 题 pilot 中，Full System 在错因定位、证据支持、幻觉控制上优于 Direct LLM 与 OCR+LLM。

正式阶段成功：

- 300-500 题主集 + 外部集上，有显著统计优势。
- 各模块消融有清晰贡献。
- 失败案例诚实且可解释。
- 代码、日志、标注协议可复现。

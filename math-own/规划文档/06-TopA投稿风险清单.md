# Top A 投稿风险清单

更新时间：2026-05-15

## 当前总评

方向有研究价值，但当前材料不能直接作为 Top A 实证论文投稿。更准确的状态是：

> 研究方案 + 原型设计 + 部分黑盒测试材料 + 待真实实验验证的预实验稿。

最大的风险不是选题，而是实验真实性、指标定义和过度承诺。

## 严重风险

| 风险 | 严重程度 | 修复方式 |
| --- | --- | --- |
| 把示例表格写成真实结果 | 严重 | 全部改成“预实验/示例结果”，正式稿用真实日志替换 |
| 50 题支撑强结论 | 严重 | 50 题只作 pilot，正式主集 300-500+ |
| 测试题互相检索 | 严重 | 建独立 evidence corpus |
| 指标伪定义 | 严重 | MRR/NDCG/Accuracy 必须按标准定义和人工标注计算 |
| 同模生成与同模评估 | 中高 | LLM Judge 只作辅助，关键结论用人工/符号验证 |
| Top A 表述过度 | 中高 | 改为“具备打磨潜力”，不写“已满足” |
| 方法细节不足 | 中高 | 补公式树、关系边、路由规则、验证失败机制 |
| 参考文献与数据集命名混乱 | 中 | 修正 MATH-Vision/GSM8K/ChartQA 用途 |

## 必须改写的表达

| 原表达 | 改写 |
| --- | --- |
| 实验证明本方法显著优于所有基线 | 预实验/示例结果显示潜在趋势，正式结论需真实实验支持 |
| 论文已经具备顶会/顶刊投稿水平 | 当前具备研究潜力，但仍需真实数据、强基线和复现材料 |
| 提升 24.3% | 提高 24.3 个百分点；相对提高 38.9% |
| 2-4 周冲 Top A | 完成可投稿强稿通常需要 8-12 周以上系统实验，Top A 还取决于原创算法贡献 |
| GraphRAG 显著优于 RAG | GraphRAG 只在关系推理或多跳任务中可能收益更高 |

## 数据与指标规则

1. 百分比提升必须区分百分点和相对提升。
2. Accuracy 必须比较模型答案与标准答案的数学等价性。
3. MRR/NDCG 必须基于人工 relevance label。
4. OCR 质量使用 CER/WER/公式识别准确率。
5. 幻觉率必须明确按样本、按 claim 还是按步骤计算。
6. 关键指标必须报告置信区间和显著性。
7. 所有表格数字必须能从 evaluation 脚本复现。

## 投稿前检查清单

### 数据

- [ ] 主测试集 300-500+ 题。
- [ ] 外部泛化集 200+ 题。
- [ ] Evidence corpus 与测试题完全分离。
- [ ] 每题有原图/PDF crop、OCR 原文、校正文、公式、条件、图形、学生步骤、标准答案、推理链。
- [ ] 双人标注与 Kappa。
- [ ] 版权和来源记录完整。

### 方法

- [ ] MPES 字段定义完整。
- [ ] 公式树或公式结构表示可复现。
- [ ] 关系边构建规则明确。
- [ ] 路由器规则或模型明确。
- [ ] 验证失败后的补检索、低置信度或拒答机制明确。
- [ ] 错因原子体系有标注手册。

### 实验

- [ ] Closed-book VLM。
- [ ] OCR + LLM。
- [ ] OCR + Text RAG。
- [ ] OCR + LaTeX RAG。
- [ ] Structured RAG。
- [ ] Graph/Hybrid RAG。
- [ ] Full Math-SEARAG。
- [ ] 消融实验。
- [ ] 题型分层。
- [ ] 成本与延迟。
- [ ] 失败案例。

## Top A 还需要的原创增强

仅将 OCR、RAG、GraphRAG、SymPy 组合，可能更像系统工程。若要冲更高水平，建议补一个更明确的算法贡献：

1. 可学习公式图检索器。
2. 验证驱动路由器。
3. 符号约束解码。
4. 错因原子对齐模型。
5. MPES Benchmark。

## 最现实的投稿策略

第一篇做系统/应用型论文：结构化证据链 + 符号验证在高中数学错题诊断中的作用。

第二篇再做算法增强论文：公式图检索器、验证驱动路由、错因原子对齐等更硬的算法点。

## 最后提醒

可以有野心，但论文里必须克制。审稿人最看重的是：数据是否真实、指标是否定义正确、对照是否公平、结论是否被日志支撑、失败是否被诚实分析、方法是否别人能复现。

# 安全合规与未成年人保护

更新时间：2026-05-15

## 基本原则

这是面向高中生的教育 AI 产品，企业级要求必须把安全、隐私、未成年人保护和内容可靠性放到产品设计一开始，而不是上线后补救。

核心原则：

- 数据最小化
- 明示授权
- 未成年人保护优先
- 关键 AI 输出可追溯
- 敏感内容可审计
- 用户可导出和删除个人数据

## 数据分类

| 数据类型 | 示例 | 敏感等级 | 处理策略 |
| --- | --- | --- | --- |
| 账号数据 | 手机号、邮箱、昵称 | 高 | 加密存储、最小访问 |
| 未成年人身份数据 | 年级、学校、班级 | 高 | 明示授权、权限隔离 |
| 学习数据 | 错题、步骤、报告 | 高 | 租户隔离、访问审计 |
| 原始图片/PDF | 试卷、手写内容 | 高 | 对象存储加密、生命周期管理 |
| 模型日志 | prompt、输出、证据 | 中高 | 脱敏、分级访问 |
| 支付数据 | 订单、套餐 | 高 | 不存储完整支付敏感信息 |
| 运营数据 | 活跃、留存、点击 | 中 | 聚合分析优先 |

## 未成年人保护要求

### 授权机制

- 学生注册时需要明确年龄段。
- 涉及个人学习报告、家长报告、学校/老师绑定时，需要可验证授权。
- 学校/机构批量导入学生时，需要租户侧承诺已获得授权。

### 内容保护

AI 输出不能包含：

- 侮辱、羞辱、贬低学生的表达
- 过度焦虑和夸大提分承诺
- 与学习无关的诱导付费话术
- 不适合未成年人的内容
- 泄露其他学生隐私的信息

### 产品表达

系统应避免说：

- “你太笨了”
- “你一定能提多少分”
- “不买就会落后”

建议表达：

- “这一步主要是端点比较遗漏，可以通过 3 道同类题巩固。”
- “本周定义域意识相关错误减少，说明复盘有效。”

## AI 安全控制

### 输入侧

- 图片 OCR 前做文件类型和大小校验。
- 防止用户上传无关、违法或隐私过强内容。
- 对含身份证、手机号等敏感信息的图片提示遮挡或自动脱敏。

### 输出侧

- 所有讲解必须引用 evidence_id。
- 无证据支持时降低置信度或拒答。
- 数学结论高风险时必须经过验证层。
- 变式题必须有答案和验算记录。
- 家长报告避免诊断式心理标签。

### 模型调用侧

- API key 放在服务端密钥管理，不进入前端。
- 不把用户敏感数据发送给不可信第三方模型。
- 第三方模型调用记录 provider、用途、数据类型和用户授权状态。
- 支持企业客户选择私有化或专属模型配置。

## 权限模型

| 角色 | 权限 |
| --- | --- |
| 学生 | 查看和管理自己的题目、报告、档案 |
| 家长 | 查看授权学生的周报和建议，不默认查看全部原始题目 |
| 老师 | 查看班级内授权学生的聚合与个人学习数据 |
| 教研员 | 查看匿名化题目和标注任务 |
| 机构管理员 | 管理本机构班级、老师和套餐 |
| 系统管理员 | 管理平台配置，不默认查看明文学习内容 |

## 审计日志

必须记录：

- 登录与权限变更
- 数据导出与删除
- 老师/家长查看学生报告
- 管理员访问学习数据
- 模型输出被人工修改
- 高风险内容审核通过或驳回

审计日志字段：

- actor_id
- actor_role
- tenant_id
- action
- resource_type
- resource_id
- timestamp
- ip
- user_agent
- result

## 数据生命周期

| 数据 | 默认保留 | 删除策略 |
| --- | --- | --- |
| 原始图片/PDF | 6-12 个月 | 用户可手动删除，企业可配置 |
| OCR 中间结果 | 12 个月 | 随原题删除 |
| MPES Schema | 业务需要期间 | 可脱敏后用于研究/模型改进 |
| 学习报告 | 长期 | 用户可导出和删除 |
| 模型日志 | 3-6 个月 | 脱敏归档 |
| 审计日志 | 1-3 年 | 按合规要求保留 |

## 合规清单

- [ ] 隐私政策说明收集哪些学习数据。
- [ ] 用户协议说明 AI 输出仅作学习辅助。
- [ ] 未成年人保护条款单独成章。
- [ ] 支持数据导出和删除。
- [ ] 支持家长/学校授权管理。
- [ ] 第三方模型和 OCR 服务在隐私政策中披露。
- [ ] 生产环境开启 HTTPS。
- [ ] 数据库和对象存储加密。
- [ ] 管理后台启用 MFA。
- [ ] 关键操作有审计日志。

## 论文与产品的合规边界

论文数据集不能直接使用真实学生个人数据。若使用真实学生错题：

- 必须脱敏。
- 必须去除学校、姓名、班级、联系方式。
- 必须获得授权或使用公开/自建数据。
- 论文中不要展示可识别个人身份的原图。

## 企业级验收

企业客户上线前至少完成：

- 隐私影响评估。
- 数据流图。
- 权限矩阵。
- 安全测试。
- 内容安全测试。
- 未成年人保护流程测试。
- 数据删除演练。

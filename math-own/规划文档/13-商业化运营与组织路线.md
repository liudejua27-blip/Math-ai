# 商业化运营与组织路线

更新时间：2026-05-15

## 商业定位

产品核心卖点：

> 不卖“答案”，卖“诊断、陪练、提分路径和长期思维档案”。

这和普通拍照搜题的差异在于：

- 普通搜题解决“这题答案是什么”。
- 你的产品解决“我为什么错、怎么补、是否掌握、下周练什么”。

## 市场切入策略

### 第一阶段：高三导数/函数专题

原因：

- 错因复杂，最能体现诊断价值。
- 家长和学生付费意愿更强。
- 符号验证可以发挥作用。
- 容易设计变式题。

### 第二阶段：高考核心专题

扩展：

- 圆锥曲线
- 数列
- 立体几何
- 概率统计
- 三角函数
- 不等式与参数问题

### 第三阶段：老师/机构场景

进入班级和教培机构：

- 班级错因热力图
- 分层作业
- 教研题库
- 学生报告

## 目标客户

| 客户 | 付费理由 | 触达方式 |
| --- | --- | --- |
| 高三学生 | 高考冲刺、错题复盘提效 | 内容号、社群、同学推荐 |
| 家长 | 看懂孩子薄弱点 | 周报样例、家长社群 |
| 数学老师 | 减少批改和统计成本 | 试点班级、教研合作 |
| 教培机构 | 提升个性化服务能力 | SaaS 销售、私有题库合作 |
| 学校 | 班级学情分析 | 区域试点、课题合作 |

## 商业模式

| 模式 | 说明 | 适合阶段 |
| --- | --- | --- |
| C 端订阅 | 月费/年费解锁诊断、变式、档案 | MVP 后 |
| 高考专题包 | 导数/圆锥/数列专项 | 早期变现 |
| 按次付费 | 免费额度后按题付费 | 拉新期 |
| 教师 SaaS | 按老师/班级收费 | Pilot 后 |
| 机构版 | 按校区/学生数收费 | 企业版 |
| 教研服务 | 定制题库和错因体系 | 机构合作 |

## 冷启动策略

### 内容冷启动

输出高质量内容：

- 一张图看懂这道导数题为什么错
- 高考数学 10 类高频错因
- 端点比较为什么总被忽略
- 参数分类讨论专项拆解

内容目标不是泛泛讲题，而是展示“错因诊断能力”。

### 用户冷启动

1. 找 30-50 名高三学生内测。
2. 每人上传 10 道错题。
3. 收集是否看懂、是否愿意继续用、是否愿意付费。
4. 找 3-5 名数学老师复核错因诊断。

### 数据冷启动

从一个专题开始，不要一开始覆盖全高中数学。

建议第一批：

- 导数题 50 道
- 函数题 50 道
- 圆锥曲线 50 道

## 运营指标

| 阶段 | 核心指标 |
| --- | --- |
| 内测 | 完成诊断率、主观有用评分、复用率 |
| MVP | 次日留存、7 日再次上传、报告打开率 |
| 付费验证 | 付费转化、退款率、客单价 |

## 融资/商业故事线

一句话：

> 用结构化证据链和错因原子，把 AI 从“给答案的工具”升级为“可验证的数学思维教练”。

故事结构：

1. 高考数学错题复盘是刚需，但传统搜题只给答案。
2. 学生真正需要知道为什么错、怎么练、是否掌握。
3. 我们用 MPES 证据链和符号验证保证讲解可靠。
4. 用错因原子和思维档案形成长期个性化数据壁垒。
5. 从 C 端错题诊断切入，扩展到老师端和机构 SaaS。

## 风险与应对

| 风险 | 应对 |
| --- | --- |
| OCR 识别不稳 | 用户确认 + 低置信度标红 + 人工修正数据回流 |
| 讲解错误影响信任 | 证据引用 + 符号验证 + 低置信度拒答 |
| 获客成本高 | 从专题内容和老师试点切入 |
| 同质化竞争 | 做错因原子和思维档案，不做单纯搜题 |
| 教研成本高 | 先聚焦导数/函数，逐步扩展 |
| 合规风险 | 未成年人保护、授权、脱敏、审计 |

## 12 个月路线

| 时间 | 目标 |
| --- | --- |
| 0-1 月 | 10 题 MVP、错因诊断 Demo |
| 1-2 月 | 50 题 pilot、学生内测 |
| 2-3 月 | 导数/函数专题可用版本 |
| 3-4 月 | 付费验证、周报、思维档案 |
| 4-6 月 | 老师端批量导入和班级报告 |
| 6-9 月 | 扩展核心专题，机构试点 |
| 9-12 月 | 企业版、多租户、规模化运营 |

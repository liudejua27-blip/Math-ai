# 企业级系统架构蓝图

更新时间：2026-05-15

## 架构目标

企业级架构必须同时满足：

- 单题诊断低延迟
- 批量试卷异步处理
- 多租户隔离
- 模型可替换
- 数据可追溯
- 实验可复现
- 安全合规
- 成本可控

## 总体架构

```text
客户端层
  学生端 App/Web
  家长端 Web/App
  老师端 SaaS
  教研后台
  管理后台

API 网关层
  鉴权、限流、路由、审计、灰度

业务服务层
  用户与组织服务
  题目服务
  证据 Schema 服务
  错因诊断服务
  变式训练服务
  学习档案服务
  报告服务
  支付与套餐服务

AI 编排层
  OCR/公式识别适配器
  MPES 构建器
  检索路由器
  LLM 生成器
  符号验证器
  错因原子分类器
  变式题生成器
  安全审核器

数据层
  PostgreSQL
  对象存储
  向量数据库
  图数据库或关系边表
  Redis
  日志/事件仓库

MLOps/评估层
  Prompt 版本管理
  数据集版本管理
  实验运行记录
  指标评估
  模型灰度
  失败样本回流
```

## 服务拆分

| 服务 | 主要职责 | 数据 |
| --- | --- | --- |
| Identity Service | 用户、角色、组织、租户 | users, orgs, roles |
| Problem Service | 题目上传、状态流转、文件管理 | problems, assets |
| Parsing Service | OCR、公式识别、低置信度字段 | parse_jobs, parse_outputs |
| Evidence Service | MPES 生成、证据对象、关系边 | evidence_nodes, evidence_edges |
| Retrieval Service | 文本/公式/图检索 | indexes, retrieval_logs |
| Reasoning Service | LLM 解题、步骤生成、错因解释 | generation_runs |
| Verification Service | SymPy/Wolfram/规则验证 | verification_runs |
| Diagnosis Service | 步骤对齐、错因原子分类 | diagnoses, atoms |
| Variant Service | 变式题生成、答案验算 | variants |
| Profile Service | 学生能力画像、掌握度 | mastery_profiles |
| Report Service | 周报、班级报告、导出 | reports |
| Admin Service | 审核、配置、套餐、审计 | admin_configs |

## 同步与异步任务

### 同步任务

适合用户等待：

- 登录鉴权
- 查看历史报告
- 获取已有分析结果
- 小题文本分析

### 异步任务

必须进入任务队列：

- PDF 批量解析
- 大图 OCR
- 批量班级报告
- 多模型评测
- 大规模变式题生成
- 数据集重建索引

推荐：FastAPI + Celery/RQ + Redis/RabbitMQ。

## 核心数据流

1. 用户上传题目图片。
2. Problem Service 写入题目记录和对象存储地址。
3. Parsing Service 生成 OCR/LaTeX/置信度。
4. 用户确认低置信度字段。
5. Evidence Service 生成 MPES。
6. Retrieval Service 根据路由取证据。
7. Reasoning Service 输出结构化解法。
8. Verification Service 验证关键步骤。
9. Diagnosis Service 对齐学生步骤并归因。
10. Variant Service 生成并验证变式题。
11. Profile Service 更新能力画像。
12. Report Service 生成个人/班级报告。

## 多租户设计

| 维度 | 策略 |
| --- | --- |
| 数据隔离 | 所有核心表带 tenant_id |
| 文件隔离 | 对象存储按 tenant 分桶或前缀 |
| 权限隔离 | RBAC + 资源级权限 |
| 模型配置 | 支持租户级模型与 prompt 配置 |
| 审计 | 所有敏感操作写 audit_logs |

## 模型可替换设计

AI 调用必须走适配器层：

```text
ModelAdapter
  OCRAdapter
  FormulaOCRAdapter
  LLMAdapter
  EmbeddingAdapter
  RerankAdapter
  VerificationAdapter
```

每次调用记录：

- provider
- model_name
- prompt_version
- input_hash
- output_hash
- latency
- token_usage
- cost
- error_code

## 企业级非功能指标

| 指标 | 目标 |
| --- | --- |
| 单题文本诊断 P95 | 15 秒以内 |
| 单题图片诊断 P95 | 60 秒以内，复杂题异步 |
| 系统可用性 | 99.5% 起步，企业版 99.9% |
| 任务可追踪 | 100% job 有状态和日志 |
| 关键输出可追溯 | 100% 带证据 ID 和模型版本 |
| 数据恢复 | 每日备份，关键表可回滚 |
| 成本监控 | 单题 token/OCR/验证成本可见 |

## 灰度与回滚

- Prompt 按版本发布。
- 模型按租户/用户比例灰度。
- 每个版本有自动评测门禁。
- 失败率、延迟、成本超过阈值自动回滚。
- 高风险输出进入人工审核队列。

## 企业级架构验收

- 任意一题能追踪完整链路：上传、解析、确认、Schema、检索、生成、验证、诊断、报告。
- 任意一次模型输出能追踪到模型名、prompt、配置和输入证据。
- 任意租户数据不会被其他租户访问。
- 任意失败任务可重试、可诊断、可恢复。

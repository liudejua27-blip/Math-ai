# 产品方案：高考数学私人思维教练

更新时间：2026-05-15

## 产品定位

产品不定位为“拍照搜题”，而定位为：

> 高考数学私人思维教练。

核心价值不是“告诉学生答案”，而是“指出学生错在哪一步、为什么错、对应哪个数学能力短板、下一题应该怎么练”。

## 用户痛点

| 用户 | 痛点 | 产品回应 |
| --- | --- | --- |
| 高中生 | 只知道答案，不知道自己为什么错 | 错误步骤定位 + 错因原子诊断 |
| 高三学生 | 时间紧，刷题多但复盘弱 | 高价值错因闭环 + 变式训练 |
| 家长 | 看不懂孩子数学到底弱在哪里 | 每周错因报告与建议话术 |
| 老师 | 批量分析错题成本高 | 班级错因热力图与个性化作业 |
| 教培机构 | 需要个性化辅导效率 | 老师端 SaaS 与错因诊断引擎 |

## 核心闭环

1. 学生上传错题/试卷/手写步骤。
2. 系统识别题干、公式、图形和学生步骤。
3. 低置信度公式、区间、上下标、根号、分式、正负号标红，学生确认。
4. 系统生成 MPES 结构化证据。
5. 系统生成标准解题路径，每一步带证据 ID。
6. 系统对齐学生步骤和标准步骤，定位错误发生点。
7. 系统归因到错因原子。
8. 系统调用符号验证，检查关键计算和推导。
9. 系统生成 3 道同原子变式题。
10. 学生再次作答，系统判断是否掌握。
11. 系统更新个人数学思维档案。

## MVP 必做功能

### 1. 拍照/上传错题

支持图片、PDF、纯文本输入。第一版优先支持图片和文本，PDF 可以只支持单页或裁剪页。

### 2. OCR + 用户确认

数学 OCR 不能直接相信。必须让用户确认低置信度字段：

- 上下标
- 分式层级
- 根号范围
- 区间端点
- 正负号
- 参数范围
- 几何条件

### 3. 学生步骤输入

第一版不强求手写步骤自动识别，可先允许学生手动输入或拍照后人工修正。错因诊断必须依赖学生自己的解题过程，否则只能给普通讲解。

### 4. 证据链解题

每一步解题必须引用证据：

- C：题干条件
- F：公式
- G：图形线索
- S：学生步骤
- A：知识原子
- V：验证结果
- P：正确路径
- E：错误定位

### 5. 错因原子诊断

不要只说“粗心”或“不会导数”。应归因到更具体的能力原子：

- 审题漏条件
- 公式记错
- 代数变形错误
- 定义域意识弱
- 分类讨论缺失
- 端点比较遗漏
- 参数分析弱
- 数形结合弱
- 几何辅助线意识弱
- 推导链断裂

### 6. 符号验证

第一版优先接入 SymPy：

- 表达式等价
- 求导验证
- 方程代入验证
- 简单不等式/数值采样
- 变式题答案检查

更复杂题目后续接 Wolfram 或人工验证。

### 7. 变式题生成

不要让 AI 随机编题。应基于：

母题模板 + 参数变化 + 难度梯度 + 符号/数值验算

默认生成 3 道：

1. 基础同原子题
2. 中等综合题
3. 高考压轴风格题

### 8. 个人数学思维档案

记录：

- 学生常错原子
- 错误发生步骤分布
- 题型掌握度
- 变式题通过率
- 本周进步曲线
- 下周训练建议

## 第一版页面设计

### 首页

主功能区域不是营销落地页，而是直接上传错题：

- 上传错题
- 输入解题步骤
- 查看错因雷达
- 查看思维档案

### 错题分析页

三栏布局：

| 左侧 | 中间 | 右侧 |
| --- | --- | --- |
| 原题图片、题干、公式 | 学生步骤 vs 正确步骤 | 证据链、错因、验证结果 |

### 错因雷达页

展示本题错因和历史错因：

- 参数分类讨论缺失：高风险
- 端点比较遗漏：中风险
- 代数变形错误：低风险

### 思维档案页

展示长期画像：

- 能力原子掌握度
- 错因趋势
- 章节表现
- 题型表现
- 变式题通过率

## 暂缓功能

第一版暂缓：

- 完整老师端后台
- 家长端复杂报告
- 大规模题库系统
- Lean 形式化证明
- 纯 AI 生图海报
- 复杂社交对战功能

可视化学习卡片可以做，但应使用 JSON -> SVG / Canvas / Mermaid，而不是直接用图片生成模型画数学公式。

## 商业模式

| 模式 | 说明 |
| --- | --- |
| 订阅制 | 月费/年费用错题诊断与思维档案 |
| 按次付费 | 免费额度后按题付费 |
| 高考冲刺包 | 按专题与预测题打包 |
| 老师端 SaaS | 班级错因热力图和作业推荐 |
| 机构合作 | 教培机构接入错因诊断引擎 |
| 家长报告 | 每周生成学习报告和建议话术 |

## 产品成功指标

| 指标 | 第一阶段目标 |
| --- | --- |
| 错误步骤定位准确率 | 70%+ pilot |
| 错因原子诊断教师一致率 | 70%+ pilot |
| 变式题通过率提升 | 有明显趋势即可 |
| 单题分析时长 | 30-90 秒内 |
| 用户愿意二次上传比例 | 30%+ 内测 |
| 学生主观帮助评分 | 4/5 以上 |

## 最小可演示 Demo

Demo 只需要覆盖一类高价值题型，例如导数或函数最值题：

1. 用户上传一道导数错题。
2. 输入自己的解题步骤。
3. 系统识别“定义域/端点/参数分类”相关证据。
4. 系统指出错误发生在第几步。
5. 系统解释错因原子。
6. 系统给出通过 SymPy 验证的正确推导。
7. 系统生成 3 道同原子变式题。

这个 Demo 跑通后，再扩展到圆锥曲线、数列、概率统计、立体几何。

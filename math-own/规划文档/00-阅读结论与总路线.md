# 阅读结论与总路线

更新时间：2026-05-15

## 核心阅读结论

我阅读并整合了 `论文/论文max.docx`、`论文/给我安排的计划.docx`、根目录下 Math-SEARAG 三份方案/实验/风险文档、`主要思路.docx`、`核心.docx`、`gpt论文错误.docx`、产品目录材料、CSV/XLSX 黑盒测试数据以及 `math/`、`文献top/` 下的文献主题。结论很明确：项目方向有价值，但必须从“想法和示例稿”推进到“可运行、可标注、可复现、可投稿”的真实闭环。

你的项目不应被写成普通数学 App，也不应写成泛泛的多模态 RAG。最稳妥的定位是：

> 面向高中数学错题诊断/数学试卷问答的公式感知结构化证据增强 RAG 与符号验证系统。

产品端叫“高考数学私人思维教练”；论文端继续使用 Math-SEARAG 或 FASER-Math。

## 统一主线

### 产品主线

上传错题或试卷后，系统不是先给答案，而是：

1. 识别题干、公式、图形线索和学生解题步骤。
2. 标红低置信度的公式、条件、上下标、区间、根号、分式。
3. 让学生确认关键字段。
4. 构建 MPES 结构化证据。
5. 生成带 evidence_id 的标准解题路径。
6. 对齐学生路径与正确路径，定位错误发生点。
7. 归因到错因原子。
8. 调用 SymPy/Wolfram/规则验证关键推导。
9. 生成 3 道同原子变式题。
10. 根据再次作答更新数学思维档案。

### 论文主线

论文研究问题是：

> 公式感知解析、结构化证据链、错因原子图谱和符号验证能否降低数学问答中的公式幻觉、条件误用、推理跳步，并提升错因诊断准确率？

推荐中文题目：

> 面向高中数学错题诊断的公式感知结构化证据增强 RAG：基于错因原子与符号验证的幻觉缓解方法研究

## 关键风险判断

| 风险 | 结论 | 修复动作 |
| --- | --- | --- |
| 示例表格被当成真实结果 | 高风险 | 全部改为预实验/示例，正式稿用真实日志替换 |
| 50 题支撑强结论 | 高风险 | 50 题只作 pilot，正式主集 300-500 题以上 |
| 测试题互相检索 | 高风险 | 必须建设独立 evidence corpus |
| 指标定义不严谨 | 高风险 | Accuracy、MRR、NDCG、幻觉率都要有可操作定义 |
| Top A 表述过满 | 高风险 | 改为“具备打磨潜力”，不承诺已满足 |
| 脚本路径和依赖不稳 | 中风险 | 统一 PROJECT_ROOT，先跑通小闭环 |

## 三阶段路线

### 第一阶段：真实 MVP，1-2 周

目标：5-10 道题跑通完整链路。

交付物：

- Python 项目骨架
- OCR/公式识别原始输出
- 5-10 个人工校验 MPES
- 一版错因原子分类
- SymPy 验证 demo
- 运行日志和失败清单

### 第二阶段：Pilot Study，2-4 周

目标：50 题验证趋势，发现系统真实短板。

交付物：

- 50 题 pilot set
- 标注规范 v0.2
- 错因原子体系 v0.2
- Baseline A/B/C/Full 的 pilot 结果
- 错误分析报告

### 第三阶段：正式实验，5-12 周

目标：300-500 题主测试集 + 外部泛化集 + 强基线。

交付物：

- 主数据集
- 独立 evidence corpus
- 双人标注和 Kappa
- 消融实验
- 统计检验
- 失败案例图表
- 投稿前论文稿

## 优先级排序

| 优先级 | 任务 | 原因 |
| --- | --- | --- |
| P0 | 统一论文定位和风险表述 | 防止学术诚信与投稿风险 |
| P0 | 定义 MPES 与标注规范 | 产品和论文共同底座 |
| P0 | 跑通 5-10 题 MVP | 真实闭环比完整大系统更重要 |
| P1 | 构建独立 evidence corpus | 防止答案泄漏和伪相关 |
| P1 | 做 50 题 Pilot | 找出系统真实短板 |
| P1 | 做错因原子图谱 v0.1 | 产品差异化核心 |
| P2 | 扩到 300-500 题 | 支撑正式论文结论 |
| P2 | 做 App 内测 | 验证真实用户价值 |

## 最重要的执行原则

- 先做真实闭环，不先堆大系统。
- 先做证据库和标注规范，不用测试题互相检索冒充 RAG。
- 先证明错误步骤定位和错因原子诊断，再做复杂商业功能。
- 论文中所有数字必须能从日志、脚本和标注文件复现。
- Top A 不是靠标题冲上去，而是靠数据、基线、消融、统计检验、失败分析和可复现性撑起来。

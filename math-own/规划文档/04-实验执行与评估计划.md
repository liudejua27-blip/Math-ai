# 实验执行与评估计划

更新时间：2026-05-15

## 实验总原则

先跑通最小闭环，再扩样本；先固定 Schema、日志和评估格式，再追求大规模结果。

当前黑盒测试数据和示例表格可以用来诊断工程流程，但不能直接写成正式 Top A 论文结果。正式结果必须来自真实题库、独立证据库、人工标注、强基线和统计检验。

## 数据分层

| 数据层 | 规模 | 用途 |
| --- | ---: | --- |
| MVP Set | 5-10 题 | 跑通全链路，验证工程可行性 |
| Pilot Set | 50 题 | 修正 Schema、指标、错因原子和失败类型 |
| Main Test Set | 300-500 题 | 正式论文主结果 |
| External Set | 200-500 题 | 外部泛化验证 |
| Evidence Corpus | 独立构建 | 教材知识点、定理卡、公式库、标准例题，不能使用测试题答案 |

## Baseline 设计

| 方法 | 输入 | 核心模块 | 目的 |
| --- | --- | --- | --- |
| Direct LLM / VLM | 原图或题目文本 | 不检索、不结构化 | 参考下限 |
| OCR + LLM | OCR 文本 | 直接生成 | 测试 OCR 压扁后的能力 |
| OCR + Text RAG | OCR 文本 + 独立证据库 | 文本检索 | 最弱 RAG 基线 |
| OCR + LaTeX RAG | OCR 文本 + LaTeX | 文本与公式检索 | 测公式信息增益 |
| Structured RAG | MPES 字段 | 字段检索，无验证 | 测结构化中间表示收益 |
| Graph/Hybrid RAG | MPES + 关系边 | 统一图检索 | 与 Selective 策略比较成本收益 |
| Math-SEARAG w/o Verification | 完整生成，无验证 | 去掉验证层 | 测验证贡献 |
| Full Math-SEARAG | Parsing + MPES + Route + Verify | 完整系统 | 主方法 |

## 评价指标

### 解题正确性

| 指标 | 定义 |
| --- | --- |
| Answer Accuracy | 最终答案与标准答案数学等价的比例 |
| Step Accuracy | 关键推理步骤正确比例 |
| Formula Accuracy | 公式识别和使用是否正确 |
| Calculation Accuracy | 计算过程是否正确 |

### 证据与检索

| 指标 | 定义 |
| --- | --- |
| Evidence Recall@K | Top-K 是否覆盖标准解法所需证据 ID |
| Precision@K | Top-K 中人工标注相关证据比例 |
| MRR | 第一个相关证据的倒数排名，必须基于 relevance label |
| NDCG | 排名质量，必须基于人工相关性等级 |
| Formula Grounding Accuracy | 答案中公式是否来自 Schema 或可合法推出 |
| Condition Grounding Accuracy | 使用条件是否真实存在且绑定正确 |
| Relation Consistency | 推理路径是否沿关系边或合法证据链展开 |

### 幻觉控制

| 指标 | 定义 |
| --- | --- |
| Unsupported Step Rate | 无证据支撑步骤数 / 总步骤数 |
| Formula Hallucination Rate | 编造公式或错误公式使用比例 |
| Condition Misuse Rate | 使用不存在或错误条件比例 |
| Condition Omission Rate | 关键条件遗漏比例 |
| Reasoning Jump Rate | 推理跳步或未说明中间依据比例 |
| Overall Hallucination Rate | 明确说明按样本、按 claim 还是加权错误类型计算 |

### 错因诊断

| 指标 | 定义 |
| --- | --- |
| Error Localization Accuracy | 错误步骤定位准确率 |
| Misconception Classification Accuracy | 错因类型分类准确率 |
| Atom Diagnosis Accuracy | 数学原子诊断准确率 |
| Teacher Agreement Rate | 与教师标注一致率 |

### 系统指标

| 指标 | 定义 |
| --- | --- |
| Latency | 端到端延迟 |
| Token Cost | 单题 token 成本 |
| Verification Pass Rate | 验证通过率 |
| Re-retrieval Rate | 补检索触发率 |
| Human Confirmation Rate | 用户确认字段比例 |

## 消融实验

| 消融 | 目的 |
| --- | --- |
| 去掉公式感知解析 | 验证公式识别贡献 |
| 去掉 MPES Schema | 验证结构化证据贡献 |
| 去掉自适应路由 | 验证不同题型差异路径贡献 |
| 去掉 Selective Graph Retrieval | 验证复杂题图检索贡献 |
| 去掉符号验证 | 验证计算和逻辑幻觉控制贡献 |
| 去掉错因原子 | 验证错因诊断贡献 |
| 去掉人工确认 | 验证 human-in-the-loop 的必要性 |
| 只用 LLM Judge | 检查同模评估是否高估 |

## 统计检验

- 准确率类：McNemar test 或 paired bootstrap。
- 连续指标：paired bootstrap / permutation test。
- 多方法比较：报告均值、标准差、95% CI。
- 人工标注：Cohen's Kappa，目标 0.75+，核心幻觉标签最好 0.8+。
- 分题型报告：普通文字题、公式密集题、几何图形题、多步推导题。

## 阶段计划

| 阶段 | 时间 | 目标 | 验收标准 |
| --- | --- | --- | --- |
| Phase 0 | 1-3 天 | 降风险修稿 | 所有未验证数字改为示例或预实验 |
| Phase 1 | 1 周 | 5-10 题 MVP | 每题有输入、Schema、输出、验证、日志 |
| Phase 2 | 2-4 周 | 50 题 Pilot | 有错误分析和 pilot 表格 |
| Phase 3 | 5-8 周 | 300-500 题主集 | 双人标注、证据库、Kappa |
| Phase 4 | 8-10 周 | 跑全部实验 | baseline、消融、统计检验完成 |
| Phase 5 | 10-12 周 | 论文重写 | 第 5 章替换真实结果，补失败分析 |

## 当前黑盒测试数据的用法

已有 `math_searag_blackbox_test_data.csv` 覆盖 C1-C8：

- 清晰单题
- 中文应用题
- 公式密集题
- 含表格/图形/版面块
- 模糊噪声
- 证据库缺失
- 答案泄漏探针
- API 故障

这些条件适合作为工程鲁棒性测试集。建议保留为 black-box smoke test，而不是正式论文主结果。

## 禁止做法

- 不用测试题互相检索作为 evidence corpus。
- 不用平均相似度替代 Accuracy。
- 不用“有返回结果”替代 Hit@1。
- 不用答案字符串出现率替代解题正确率。
- 不用 Simulated Generation 冒充真实模型生成。
- 不用同一个 LLM 同时生成和作为唯一评估者。
